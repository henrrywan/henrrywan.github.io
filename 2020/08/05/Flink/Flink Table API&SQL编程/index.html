动态表（Dynamic Tables）

动态表是Flink在Table API和SQL中的核心概念。我们所熟悉的表一般是用来做批处理，面向的是固定的数据集，可以认为是“静态表”，而动态表里面的数据会不断变化。
由于数据在不断变化，因此基于它定义的SQL查询也不可能执行一次就得到结果，这样一来，动态表的查询永远不会停止，一直会随着新数据的到来而继续执行，这样的查询被称为“持续查询（Continuous Query）”。
持续查询可以分为以下三步：

  • 流被转换成动态表
  • 对动态表进行持续查询生成新的动态表
  • 生成的动态表被转换成流

SQL持续查询可以分为两类：

  • 更新（Update）查询
  • 追加（Append）查询

动态表转换成流支持三种编码方式：

  • 仅追加（append-only）流
  • 撤回（retract）流
  • 更新插入（upset）流

窗口表值函数

从 1.13 版本开始，Flink 开始使用窗口表值函数（Windowing table-valued functions，Windowing TVFs）来定义窗口。
目前提供以下几个窗口TVF：
滚动窗口（Tumbling Windows）

滑动窗口（Hop Windows，跳跃窗口）

累积窗口（Cumulate Windows）

会话窗口（Session Windows，目前尚未完全支持）
