Flink常用的DataSet和DataStream API

前言

在这一章节中，我们主要学习Flink中常用的DataSet和DataStream API。

DataSet和DataStream的区别

虽然Flink一直强调自己的“流批一体”的特性，但实际上，到目前为止，Flink在处理流批数据上仍然需要编写两套代码，即使用DataSet API处理批数据，使用DataStream API处理流数据。

在Flink的源代码中，可以在flink-java这个模块中找到所有关于DataSet的核心类，在flink-streaming-java中可以找到所有关于DataStream的核心类。DataSet API和DataStream API高度相似。由于Flink更多的是偏向于实时计算领域，后续示例我们更多的会以DataStream API的形式进行展示。

对于DataSet而言，Source主要来源于文件、表或者Java集合；对于DataStream而言，Source主要来源于消息中间件，比如Kafka等。

常用算子

Map

转换：DataStream → DataStream
描述：采用一个数据元并生成一个数据元。
示例：将输入流的值加倍。

FlatMap

转换：DataStream → DataStream
描述：采用一个数据元并生成零个、一个或多个数据元。
示例：将句子进行单词分割

Filter

转换：DataStream → DataStream
描述：过滤掉不需要的数据
示例：过滤掉数据流中为0的值的数据（保留函数返回true的数据元，过滤掉函数返回false的数据元）。

KeyBy

转换：DataStream → KeyedStream
描述：逻辑上将流分区为不相交的分区。具有相同Keys的所有记录都分配给同一分区。

Reduce

描述：在被Keys化数据流上滚动Reduce。（在每一个分组的KeyedStream上生效，按照用户自定义的聚合逻辑进行分组聚合。）
示例：将当前数据元与最后一个Reduce的值组合并发出新值。

原因分析：
事实上，上面描述中说min返回最小值，而minBy返回该字段中具有最小值的数据元是不太严谨的。min和minBy都会返回整个元素，只是min会根据用户指定的字段取最小值，并且把这个值保存在对应的位置，而对于其他的字段，并不能保证其数值正确。max和maxBy同理。

对于Aggregations函数，Flink帮助我们封装了状态数据，这些状态数据不会被清理，所以在实际生产环境中应该尽量避免在一个无限流上使用Aggregations函数。而且对于同一个KeyedStream，只能调用一次Aggregations函数。
