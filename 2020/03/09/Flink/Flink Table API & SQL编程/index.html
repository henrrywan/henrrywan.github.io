Flink Table API & SQL编程

前言

这一章节，我们主要学习Flink Table API & SQL编程。

原理

在Flink中，Table API和SQL都属于高级API操作（分层抽象前两层）。它们实际上是为了降低用户使用实时计算门槛而设计的一套符合标准SQL语义的开发语言。事实上，Table API和SQL被集成在了一个联合的API中。此API的核心概念是Table用作查询的输入和输出。

我们知道，在Hive中，它的底层对SQL的解析使用的是Apache Calcite，Flink同样把SQL的解析、优化和执行交给了Calcite。

下面是Flink Table & SQL的实现原理图：

从这张图中我们可以看到：
不管是批查询BatchSQL还是流查询StreamSQL，都会经过对应的转换器Calcite Parser转换成节点树SQLNode Tree，然后生成逻辑执行计划Logic Plan，逻辑执行计划在经过优化后会生成真正可以执行的物理执行计划Physical Plan，最后交给对应的DataSet或DataStream API去执行。

一个完整的Flink Table API & SQL Job也是由三部分组成：

  • Source：数据源，比如Kafka、MySQL等。
  • Transformations：这里的Transformations部分是Flink Table API & SQL支持的常用的SQL算子，比如select、groupby等。
  • Sink：一般下沉到MySQL、HBase等。

动态表

在Flink中，引入了动态表的概念。与传统的静态表相比，动态表在做SQL查询的时候，SQL会做连续查询，不会终止。比如，我们从Kafka中读取数据，这些数据会被源源不断的解析成一张不断增长的动态表，我们的SQL查询会不断生成新的动态表作为结果表。
