GitHub和Gitee创建repository

如果你希望你的站点能通过 <你的 GitHub 用户名>.github.io 域名访问，你的 repository 应该直接命名为 <你的 GitHub 用户名>.github.io。这里我们创建henrrywan.github.io的repository，便于后期访问。
同理，我们在Gitee上也需要创建一个<你的 Gitee 用户名>的Git仓库，便于后期访问。
需要注意的是，在Gitee中我们必须要开启 Gitee Pages 服务（仓库-服务里面进行设置），这样才能进行访问。

Hexo补充说明

在上面的部署中，我们已经可以通过Hexo正常发布并访问个人博客。在实际的使用中，我们可能还需要新建分类和标签，同时也需要对图片进行管理等等。这里具体说明

添加README.md

具体分为三步：

  1. 在根目录source 文件夹下新建README.md文件

  2. 在根目录的_config.yml配置文件里，找到 skip_render 关键字，添加 README.md，这一步是为了让解释器跳过渲染（默认情况下会将所有md文件转换成HTML）

  3. 使用hexo clean && hexo deploy重新发布查看
